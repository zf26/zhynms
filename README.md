# 毕业旅行协调小应用

一款专为毕业室友设计的旅行协商工具，让四个人能实时协调旅行计划——填写需求、发现冲突、投票达成共识。

## 功能特点

- **房间系统** — 创建房间获得6位房间号，分享链接即可加入，无需注册
- **需求填写** — 每人填写姓名、出发城市、预算、时间、兴趣偏好（可多选）、优先级、禁忌条件
- **冲突检测** — 自动识别预算、时长、兴趣、日期四大冲突，红色高亮标注
- **让步建议** — 基于冲突生成温和参考建议，标注"仅参考"，决策权交还用户
- **实时投票** — 支持自定义方案，系统也会根据大家兴趣自动推荐方案，实时展示投票进度
- **共识确认** — 得票 ≥ 3 票自动展示最终方案，汇总所有人的让步点

## 技术栈

纯原生 HTML + CSS + JavaScript，单文件实现，无需构建步骤，无需后端，数据存储在浏览器 LocalStorage 中。

## 使用方法

直接在浏览器中打开 `index.html` 即可使用。

### 协作流程

1. **一人创建房间** — 点击「创建新房间」，获得6位房间号
2. **分享房间号** — 将房间号或 URL 链接分享给室友
3. **各自填写需求** — 每个人点击卡片填写自己的旅行需求
4. **查看冲突与建议** — 系统自动高亮冲突，显示让步建议
5. **发起投票** — 可添加自定义方案或使用系统推荐，室友各自投票
6. **达成共识** — 得票 ≥ 3 票后确认最终方案

## 设计理念

> 不追求复杂算法最优解，支持人工收敛，保障参与感。

旅行的意义不在于去哪里，而在于和谁一起去。这个工具只是让协商过程更透明、更高效，最终的决定权始终在你们手中。
